=== §1 TRANSCRIPTION RELIABILITY (TOP PRIORITY) ===
Symptoms to fix:
(a) After a pause, recognition restarts a "new segment" and drops the first ~seconds of the next utterance.
(b) Non-native English produces many errors.
(c) Output feels chunked into disconnected boxes.

Root cause is improper Web Speech API lifecycle. Rework the speech hook (find it — likely in session.tsx or a use-speech hook; if inline, extract to artifacts/flowmind/src/lib/use-speech-recognition.ts).

Required implementation:
- Single long-lived SpeechRecognition instance per active session. Config: continuous=true, interimResults=true, maxAlternatives=1, lang from a user-selectable setting (default 'en-US').
- NEVER let it die silently. onend handler: if session still active AND not intentionally stopped → call recognition.start() immediately inside try/catch (swallow "already started" InvalidStateError). This eliminates the multi-second cold-restart gap.
- Watchdog timer: if no onresult AND no onspeechstart for 6s while session active → force a stop()+start() cycle to recover from silent hangs. Browsers (esp. Chrome) auto-terminate after ~50–60s of silence; the watchdog + onend-restart must make this seamless.
- Interim results: render the live interim transcript IMMEDIATELY in a "live line" at the bottom of the transcript (greyed/italic), updating on every onresult. Do NOT wait for isFinal to show text. This removes the perceived "verpasst den Anfang" lag.
- Commit logic (fixes chunked boxes): maintain a rolling buffer. Append final results to the buffer. Flush a transcript chunk to POST /api/transcripts only when EITHER (i) a sentence-ending punctuation/clear pause boundary is detected, OR (ii) buffer reaches ~200 chars, OR (iii) 4s elapsed since last flush with non-empty buffer. On restart/onend, flush whatever is in the interim buffer as final so nothing is lost across restarts.
- Guard against duplicate final results (Chrome sometimes re-emits): track resultIndex (event.resultIndex) and only process new result entries.
- Add a small language/accent selector in the session header (en-US, en-GB, en-IN, de-DE, plus a few common ones) that sets recognition.lang live (restart recognizer on change). Persist choice in localStorage `fm_stt_lang`. Setting the correct lang materially improves accuracy for non-native speakers — surface a one-time hint "Tip: set your spoken language for better accuracy."
- Cleanup: on session end / component unmount, set an intentionalStop flag, stop recognizer, clear watchdog + flush timers.

Acceptance: speaking with 2–3s pauses between sentences loses NO leading words; transcript reads as continuous flowing text, not fragmented cards; live interim text appears within ~300ms of speaking.
